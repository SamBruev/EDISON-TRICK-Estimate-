# Задание для дизайнер-агента · EDISON TRICK Estimate

## Контекст проекта

**Продукт:** мобильный одностраничный сайт — интерактивная смета на производство **3 песен** группы **EDISON TRICK**.  
**Исполнитель:** Sam Bruev.  
**Аудитория:** заказчик / партнёр — человек, который должен быстро понять объём работ, сроки, стоимость и условия.

**Живой сайт:** https://sambruev.github.io/EDISON-TRICK-Estimate-/  
**Репозиторий:** https://github.com/SamBruev/EDISON-TRICK-Estimate-

**Стек:** статический HTML + CSS + vanilla JS (`index.html`, `estimate.js`, `calendar.js`, `logo-scroll.js`, `nav-scroll.js`). Без фреймворков. Деплой — GitHub Pages.

---

## Текущая структура страницы

1. **Hero** — логотип, сумма, срок сдачи, фоновая лампа
2. **Навигация** — якорные ссылки по секциям
3. **Варианты записи** — чекбоксы «Бас вживую» / «Барабаны вживую», пересчёт суммы
4. **Сводная таблица** — позиции сметы + строка «Итого»
5. **Календарный график** — месяцы, события по инструментам, легенда, детали дня
6. **Ключевые даты** — чипы с датами
7. **График оплаты** — этапы и суммы
8. **Условия** — правки, ответственность сторон
9. **Результат** — что сдаётся, дедлайн
10. **Footer bar** — фиксированная сумма и срок

**Визуальная тема сейчас:** чёрный фон, «жидкое стекло» (glass), Roboto Slab, иконка лампы (`assets/symbol.png`), контрастные цвета инструментов в календаре.

## Известные боли (обратная связь заказчика)

- Раньше золотые/оливковые подложки выглядели как «зелёная хуйня» — убраны в пользу нейтрального стекла. Нужно проверить, что акценты не возвращаются в нежелательный спектр.
- Строка **«Итого»** и похожие блоки должны ощущаться как **прозрачное стекло на чёрном**, без цветной заливки.
- **Анимации** (варианты записи, рост цены) должны быть **спокойными**, не «активными».
- Индикатор «вживую» — **красная точка Record**, не золотая.
- **Фоновая лампа** — атмосферный элемент, не должна доминировать (сейчас уменьшена и размыта).
- **Календарь:** у каждого типа работ свой **контрастный цвет** — это осознанное решение, не унифицировать в серое.

## Цель задания

Провести **полный UX/UI аудит** сайта и предложить (а по возможности — реализовать) изменения, которые сделают смету:

- **гармоничнее** — единая визуальная система, ритм, отступы, типографика;
- **понятнее** — иерархия, сканируемость, логика блоков;
- **спокойнее** — без визуального шума и лишней анимации;
- **дороже** — ощущение профессионального музыкального продакшена, не «таблица в Excel».

## Ограничения (обязательно)

- Сохранить **мобильный first** — сайт в первую очередь для телефона.
- Сохранить **тёмную тему**: чёрный фон + прозрачное стекло. Без цветных «зелёно-золотых» подложек на акcentных строках.
- **Не ломать** интерактив: пересчёт суммы при «Бас/Барабаны вживую», календарь, якорная навигация.
- **Календарь:** контрастные цвета типов работ — сохранить и при необходимости только улучшить читаемость.
- **Record-точка** для live-вариантов — красная.
- Минимум зависимостей: правки в существующих HTML/CSS/JS, без React/Vue, если не согласовано отдельно.
- Не удалять и не переписывать бизнес-логику сметы без явной причины.
