# Проверка анимации обновления страницы (edge-refresh)

Аудит механики свайпа-обновления: `edge-refresh.js` + CSS-оверлей в `index.html`.
Дата: 27 июня 2026.

---

## ✅ Что работает корректно

**Распознавание жеста**
- Триггер только у правого края экрана: `EDGE = 34` px.
- Порог срабатывания `THRESHOLD = 92`, перетяг до `MAX_PULL = 128` — запас ~36 px, прогресс клампится в `setProgress` (0…1).
- Защита от ложного срабатывания при вертикальном скролле: две проверки соотношения `dy/dx` (`dx < 6 && dy > 14`, `dy > dx * 1.15 && dx < 24`).
- `preventDefault` на `touchmove` (`passive:false`) включается только при горизонтальной тяге — вертикальный скролл не блокируется.
- `isBlockedTarget` корректно исключает nav, сетку и табы календаря, поля ввода и тогглы (после правки селектора `.cal-tabs → .cal-month-tabs`).

**Коммит-фаза (тайминги выверены)**
- Трансформ `.wrap`: переход `0.55s`.
- Вспышка `edgeRefreshFlash`: `0.62s`.
- `window.location.reload()`: `620ms` (обычный) / `80ms` при `reduced-motion`.
- Все три согласованы — конец анимации не обрезается перезагрузкой.

**Прочее**
- Кольцо прогресса: `r = 18` → длина `2πr ≈ 113`, `stroke-dashoffset` сглажен переходом `0.08s`.
- `@media (prefers-reduced-motion: reduce)` отключает трансформы и анимации оверлея.
- Повторный жест во время обновления блокируется флагом `refreshing`.

## 🔴 Главная проблема плавности — рывок при отпускании без обновления

**Симптом.** Если потянуть, но отпустить, не дотянув до порога, контент страницы дёргается обратно скачком вместо плавного отката.

**Причина.** `reset(true)` мгновенно выставляет `--edge-refresh-progress: 0`. Все привязанные к прогрессу свойства — сдвиг `.wrap` на `-22px`, вуаль, луч, кромка, панель — возвращаются в ноль **без анимации**. Класс `is-snapping` навешивает переход только на `opacity/visibility` контейнера (`0.32s`), но трансформы и прогресс-зависимые значения не сглаживает.

Корень: `--edge-refresh-progress` — обычная CSS-переменная, её нельзя анимировать переходом без регистрации через `@property`. Сейчас такой попытки и нет.

---

## 💡 Рекомендации

**1. Сгладить откат (основное).**
Зарегистрировать переменную как анимируемую:

```css
@property --edge-refresh-progress {
  syntax: '<number>';
  inherits: true;
  initial-value: 0;
}
```

И при откате/снапе вешать на `html` короткий переход именно этой переменной:

```css
html.edge-refresh-settling {
  transition: --edge-refresh-progress 0.34s cubic-bezier(0.22, 1, 0.36, 1);
}
```

В `edge-refresh.js`, в ветке снапа `reset(true)`, добавлять класс `edge-refresh-settling` перед `setProgress(0)` и снимать его по таймауту. Во время самого перетягивания перехода быть не должно — палец ведёт 1:1.

Результат: сдвиг страницы, луч, кромка и кольцо поедут в ноль синхронно и плавно.

**2. Микро-оптимизация рендера.**
Добавить `will-change: transform` на `.wrap` и `.edge-refresh-beam` — меньше микро-джиттера на слабых телефонах.

---

## Итог

Логика жеста и коммит-анимация — корректны и хорошо отлажены по таймингам. Единственный реальный изъян плавности — резкий откат при отпускании без срабатывания; лечится регистрацией `@property` и переходом переменной только в фазе снапа.
